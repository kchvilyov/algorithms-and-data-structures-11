## Практическое задание 10
Необходимо реализовать мини-фреймворк, реализующий подход
MVVM, который будет содержать следующие компоненты:

- Модель: содержит данные и операции. Моделей может быть
  множество, и у них может быть любая структура;

- View Model: содержит операции над моделью. Одна модель
  может использоваться несколькими компонентами, и один компонент может
  использовать множество моделей. Если какая-либо модель изменилась, все
  работающие с ней компоненты должны узнавать об этом;

- Представление: предлагается сделать на базе Swing API.
  Необходимо сделать обёртки над компонентами JTextForm и JButton, такие, что
  изменение в одном текстовом поле приводит к обновлению информации в остальных
  полях. Если Swing кажется сложным, можно предложить свой вариант визуальной
  компоненты.

## Критерий приема
Для работы требуется Java 17.
Запуск тестов выполняется через `./gradlew test`. Задание считается выполненным, если все тесты успешно пройдены.
